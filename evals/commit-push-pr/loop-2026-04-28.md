## Remaining Weak Assertions

- [instruction_adherence] tc-01: git status is run after commit to verify success
- [instruction_adherence] tc-02: git log or git diff against base branch is used to understand full branch history

### Analysis

The post-commit `git status` assertion resisted 4 different placement strategies. The instruction exists in the skill ("Important Notes" section and Step 5's "Verify push succeeded with git status") but the model treats advisory notes as optional. Every attempt to make it mandatory either didn't improve scores or caused regressions elsewhere. This may be a fundamental limitation of how Sonnet prioritizes instructions -- the post-commit verification step gets optimized away in favor of forward progress to the push/PR steps.

The `git log/diff against base branch` assertion is partially a judge calibration issue -- the model does conceptually analyze branch history but doesn't always emit the literal commands, and the judge interprets "used to understand" as requiring visible command execution.
