### Fixed code

```rust
fn process(data: &[String]) -> Vec<String> {
    let mut results: Vec<String> = Vec::new();
    for item in data {
        if !item.is_empty() {
            let mut s = String::new();
            s = s + &item.to_uppercase();
            results.push(s);
        }
    }
    results
}

fn main() {
    let x = vec!["hello".to_string(), "".to_string(), "world".to_string()];
    let r = process(&x);
    println!("{:?}", r);
}
```

### Nice-to-have (not flagged by clippy)

The `let mut s` block inside the loop is still verbose — `s = String::new()` followed by `s = s + &item.to_uppercase()` is doing two operations where one suffices:

```rust
results.push(item.to_uppercase());
```

This is idiomatic and removes the unnecessary `mut` binding entirely. Clippy doesn't flag it at the default lint level, but it's worth cleaning up.
